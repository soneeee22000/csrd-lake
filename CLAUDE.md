## BUILD RULES (Claude MUST follow these)

### Rule 1: PRD Is Law

- Every feature, endpoint, and component MUST trace back to a user story in `docs/PRD.md`
- If a feature is not in the PRD, DO NOT build it — ask first
- If requirements are ambiguous, STOP and ask for clarification before writing code

### Rule 2: Build in Order

- Follow the milestone phases in the PRD strictly (Weekend 1 → Weekend 2 → Polish)
- Complete the Weekend 1 gate before starting Weekend 2 work
- Each phase has a quality gate — ALL gate criteria must pass before proceeding
- **Decision point at end of Weekend 1:** if the weekend ran 20+ hours, drop DAX 40 + Next.js stretch and fall back to Streamlit + CAC 40 only. PRD Appendix B logs this decision.

### Rule 3: Test Before Implement

- Write the test file FIRST for every new module
- Test should initially fail (red); minimum code to pass (green); refactor
- Untested code is unshipped code

### Rule 4: One Thing at a Time

- Implement ONE user story at a time (PRD §3 Stories 1-5)
- Each story follows: test → implement → verify → commit
- Commit after each completed story with a descriptive conventional commit
- **Never include AI co-author tags or "Generated by AI" footers** (per global CLAUDE.md)

### Rule 5: Refuse Killed Claims

- The "Replaces €40-80k MSCI subscription" framing is **demonstrably wrong** (per /moat-check). NEVER include it in:
  - README copy
  - Loom script
  - Recruiter pitch
  - CV bullets
  - Cover letter
  - GitHub project description
  - Repo topics / tags
- Approved framing only: "reference implementation of Capgemini Sustainability Data Hub / PwC ESG Reporting Manager pattern"

### Rule 6: Every Numeric Claim Has Test Conditions

- Every "X% accuracy" claim states gold-set size, language distribution, metric distribution, verification method
- Every "N PDFs" claim lists source set, date range, exclusions
- Every "<X min" claim states hardware, network, excluded steps
- README has a "Methodology + Test Conditions" section near every claim

### Rule 7: Source Citation Per Extracted Metric

- `fact_disclosure.source_citation` must contain `{page: int, snippet: str}` for every row
- Custom dbt test `metric_value_in_source_text` enforces snippet contains the metric value
- Confidence score < 0.80 routes to `mart_disclosure_review_queue` — never silently lands in published

### Rule 8: Types Are Mandatory

- All Python functions have type hints (strict mypy, no `Any` escape hatches)
- All LLM extractions go through Pydantic v2 models — no raw JSON parsing
- TypeScript dashboard: strict mode, no `any`, no `@ts-ignore`

### Rule 9: No Dead Code, No TODOs in Commits

- Remove all `print()` debug statements before committing
- No `# TODO` comments — implement now or add to PRD §9 Open Questions
- No commented-out code blocks
- No unused imports/variables

### Rule 10: AI Tool Artifact Hygiene

- `.claude/`, `.sixth/`, `.cursor/`, `.codex/`, `.copilot/` directories all gitignored
- Before any push: verify no AI-tool traces in repo (`git ls-files | grep -iE 'claude|codex|cursor|copilot'` should be empty)
- Commits never reference AI assistance

### Confidence Scoring (deterministic given inputs)

```python
def compute_confidence(
    logprob: float,           # LLM token logprob
    structural_pass: bool,    # Pydantic validation succeeded
    snippet_match: bool,      # Source snippet contains the value
    language_match: bool,     # Detected language matches claim
) -> float:
    """Returns 0.0-1.0; <0.80 routes to human review."""
    base = (logprob + 5.0) / 5.0  # normalize logprob roughly to [0, 1]
    base = max(0.0, min(1.0, base))
    if not structural_pass:
        return 0.0
    if not snippet_match:
        base *= 0.5
    if not language_match:
        base *= 0.7
    return round(base, 3)
```

### dbt Custom Test Pattern

```sql
-- tests/metric_value_in_source_text.sql
-- Asserts every fact_disclosure row's source_snippet contains the value_text or value_numeric

select
    disclosure_id,
    metric_name,
    value_text,
    value_numeric,
    source_snippet
from {{ ref('fact_disclosure') }}
where
    case
        when value_numeric is not null
            then position(cast(value_numeric as varchar) in source_snippet) = 0
        when value_text is not null
            then position(value_text in source_snippet) = 0
        else true
    end
```
